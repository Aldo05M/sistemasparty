- Botón "New Project".

- Región: la más cercana a tu ubicación.
- Espera el despliegue.

2) Obtener credenciales para frontend
- En el panel del proyecto, ve a "Settings" > "API".
- Copia:
  * URL pública (anon key): Supabase URL.
  * Anon Key (vale con clave pública).
- Reemplaza en index.html las variables:
  const supabaseUrl = 'AQUI_VA_MI_URL';
  const supabaseKey = 'AQUI_VA_MI_ANON_KEY';

3) Crear tabla "canciones" (SQL Editor)
- Ve a "SQL Editor" > "New Query".
- Ejecuta este código SQL:

```sql
-- Crear tabla canciones
CREATE TABLE canciones (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  nombre text NOT NULL,
  artista text NOT NULL,
  votos integer NOT NULL DEFAULT 1,
  inserted_at timestamp with time zone NOT NULL DEFAULT now()
);

-- Crear índices para mejor rendimiento
CREATE INDEX idx_canciones_votos ON canciones(votos DESC);
CREATE INDEX idx_canciones_nombre_artista ON canciones(nombre, artista);
```

4) Configurar RLS (Row Level Security)
- En "Table Editor" selecciona la tabla canciones.
- Activa: "Enable Row Level Security".
- O ejecuta en SQL Editor:

```sql
-- Habilitar RLS en la tabla
ALTER TABLE canciones ENABLE ROW LEVEL SECURITY;

-- Política para SELECT (todos pueden leer)
CREATE POLICY "Select all" ON canciones
  FOR SELECT USING (true);

-- Política para INSERT (todos pueden insertar)
CREATE POLICY "Insert song" ON canciones
  FOR INSERT WITH CHECK (true);

-- Política para UPDATE (solo actualizar votos)
CREATE POLICY "Update votes" ON canciones
  FOR UPDATE USING (true)
  WITH CHECK (true);
```

5) Probar con JavaScript local
- Abre index.html en tu navegador (o servidor local).
- En el panel de JS la sección de solicitud debería permitir:
  - Añadir canción con nombre+artista
  - Votar desde botón
  - Datos se guardan en la tabla canciones y se leen en #songList

6) Seguridad recomendada (después de pruebas)
- Habilita métodos de autenticación si deseas usuarios reales.
- Ajusta políticas RLS para que usuarios logueados solo puedan votar/insertar.
- Implementa validaciones adicionales para evitar spam y SQL injection.
